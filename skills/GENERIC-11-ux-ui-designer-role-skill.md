**HCI Principles to guide your work:**

```
PRINCIPLE 1: Mental Models
â”œâ”€ Users have a mental model (how they think this works)
â”œâ”€ Your interface should align with their model
â”œâ”€ NOT require them to learn new mental models
â”œâ”€ Example: Don't call it "LLM inference" call it "Ask AI"

â”œâ”€ Buttons should LOOK clickable

â”œâ”€ Disabled items should LOOK disabled
â”œâ”€ "The object should hint at how to interact"

PRINCIPLE 3: Feedback (Tell Users What Happened)
â”œâ”€ Action taken? Show confirmation
â”œâ”€ Error happened? Show clear error message
â”œâ”€ Loading? Show progress indicator
â”œâ”€ Result? Show the result immediately
â”œâ”€ NEVER: Action happens, nothing shows

PRINCIPLE 4: Constraints (Prevent Bad Actions)
â”œâ”€ Don't let user delete important thing without confirmation
â”œâ”€ Don't let them submit invalid data
â”œâ”€ Don't let them pick invalid combinations
â”œâ”€ "Make it hard to do the wrong thing"

â”œâ”€ If button A does X, button B doing something similar should work similarly
â”œâ”€ If modal A closes with Esc, modal B should too
â”œâ”€ If one form validates on blur, all should
â”œâ”€ Users learn patterns. Use them consistently.

PRINCIPLE 6: Error Prevention (Better to Prevent Than Fix)
â”œâ”€ Don't make common mistakes possible
â”œâ”€ Warn before destructive actions
â”œâ”€ Validate as you go (not just at end)
â”œâ”€ "An ounce of prevention..."

PRINCIPLE 7: Recognition vs Recall
â”œâ”€ Don't make user REMEMBER what to type
â”œâ”€ SHOW them options (dropdown, suggestions)
â”œâ”€ Recognition is easier than recall
â”œâ”€ Example: Don't ask "What's the file name?" Show list.

PRINCIPLE 8: User Control + Freedom
â”œâ”€ Let users undo/redo
â”œâ”€ Provide "emergency exits" (cancel, go back)
â”œâ”€ Don't trap them
â”œâ”€ "Users should feel in control"

PRINCIPLE 9: Visibility of System Status
â”œâ”€ User should ALWAYS know what's happening
â”œâ”€ What state are we in? (idle, loading, error, success?)
â”œâ”€ Real-time feedback (not delayed)
â”œâ”€ Example: Show which tab is active (highlighted)

â”œâ”€ Use language users understand (not jargon)
â”œâ”€ Use concepts from their world
â”œâ”€ "Speak their language, use their metaphors"
â”œâ”€ Example: "Save to Drafts" (metaphor: physical draft folder)
```
